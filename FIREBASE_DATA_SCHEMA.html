1   Architecture Overview

1.1   Data Isolation Layers

The system implements a three-tier data isolation architecture to support multi-franchise operations and demo environments simultaneously within a single Firestore database instance.

┌──────────────────────────────────────────────────────────────┐ │ CLOUD FIRESTORE │ │ │ │ ┌────────────────────────────────────────────────────┐ │ │ │ TIER 1: GLOBAL COLLECTIONS (no franchise filter) │ │ │ │ users, userPresence, notifications, franchises, │ │ │ │ plateFormats, protocolTemplates │ │ │ └────────────────────────────────────────────────────┘ │ │ │ │ ┌────────────────────────────────────────────────────┐ │ │ │ TIER 2: FRANCHISE-FILTERED COLLECTIONS │ │ │ │ Each document carries: franchiseId: String │ │ │ │ Query: WHERE franchiseId == user.franchiseId │ │ │ │ Superadmins bypass filter (read all franchises) │ │ │ │ araclar, activities, servisler, exitIslemleri, │ │ │ │ iadeIslemleri, office_operations, shuttleEntries... │ │ │ └────────────────────────────────────────────────────┘ │ │ │ │ ┌────────────────────────────────────────────────────┐ │ │ │ TIER 3: DEMO COLLECTIONS │ │ │ │ Legacy: demo_{collectionName} │ │ │ │ New: demo_environments/{userId}/{collection} │ │ │ └────────────────────────────────────────────────────┘ │ │ │ └──────────────────────────────────────────────────────────────┘

1.2   Franchise Filtering Logic

User TypeCollection PathQuery Filter Applied
Regular Production User{collectionName}WHERE franchiseId == user.franchiseId
Superadmin{collectionName}None (reads all franchises)
Legacy Demo Userdemo_{collectionName}None
New Demo Userdemo_environments/{userId}/{collectionName}None (isolated per user)

7.3   Backward Compatibility

All Swift models employ decodeIfPresent with sensible defaults to ensure forward and backward compatibility when new fields are added:

FieldDefault ValueReason
franchiseId"ch"Pre-multi-franchise documents lack this field
createdBynilLegacy documents predate user tracking
isDemoAccountfalseExisting users predate demo system
status / durum"completed" / "inProgress"Status tracking added after initial release
Optional String fieldsnil or ""Graceful handling of missing data
Optional Array fields[]Safe iteration over missing arrays
